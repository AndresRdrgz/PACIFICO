API Externa para Solicitudes
Información General

Utiliza esta API para crear solicitudes desde aplicaciones externas. Todas las solicitudes creadas vía API serán marcadas automáticamente para su identificación.

Endpoint de Creación:
POST /workflow/api/externa/solicitudes/crear/
Otros Endpoints Disponibles:
GET /workflow/api/externa/solicitudes/ - Listar solicitudes
GET /workflow/api/externa/solicitudes/{id}/ - Detalle de solicitud
GET /workflow/api/externa/solicitudes/estadisticas/ - Estadísticas
Ejemplo de Uso (cURL) 
curl -X POST {{request.scheme}}://{{request.get_host}}/workflow/api/externa/solicitudes/crear/ \
  -H "Content-Type: application/json" \
  -d '{
    "pipeline_id": PIPELINE_ID,
    "api_source": "mi_app_externa",
    "etapa_id": 3,
    "propietario_username": "admin",
    "cliente_nombre": "Juan Pérez",
    "cliente_cedula": "8-123-456",
    "cliente_telefono": "6234-5678",
    "cliente_email": "juan@email.com",
    "producto_solicitado": "Préstamo Personal",
    "monto_solicitado": 5000.00,
    "sector": "Sector Público",
    "motivo_consulta": "Consolidación de deudas",
    "observaciones": "Cliente con buen historial",
    "enlace_conversacion": "https://chat.miapp.com/conversacion/123"
  }'
Importante: Reemplaza PIPELINE_ID con el ID real del pipeline que deseas usar.
Parámetros de la API
Parámetro Tipo Requerido Descripción
pipeline_id int Sí ID del pipeline a usar (ver tabla de pipelines abajo)
api_source string Sí Identificador de tu aplicación externa
etapa_id int No ID de la etapa específica donde crear la solicitud (usa primera etapa si no se especifica)
propietario_id int No ID del usuario propietario de la solicitud
propietario_username string No Username del usuario propietario (alternativo a propietario_id)
cliente_nombre string No Nombre completo del cliente
cliente_cedula string No Cédula del cliente
cliente_telefono string No Teléfono del cliente
cliente_email string No Email del cliente
producto_solicitado string No Producto de interés del cliente
monto_solicitado decimal No Monto solicitado por el cliente
sector string No Sector laboral del cliente
motivo_consulta string No Motivo de la consulta
como_se_entero string No Cómo se enteró del servicio
observaciones string No Observaciones adicionales
enlace_conversacion string No Enlace a la conversación o chat relacionado

Identificación de Solicitudes API

Las solicitudes creadas vía API se pueden identificar fácilmente:

  • Campo "creada_via_api": Marcado como true
  • Campo "api_source": Contiene el identificador de tu aplicación
  • Campo "origen": Establecido automáticamente como "API Externa"

Filtrado en Admin: Puedes filtrar estas solicitudes en el panel de administración usando los filtros "Creada vía API" y "Fuente API".
